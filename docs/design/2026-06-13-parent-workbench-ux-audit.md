# BrainyPal 家长端工作台 UX 审计

日期：2026-06-13  
范围：RikkaHub Android 家长区域  
依据：BrainyPal PRD v0.3、当前 Compose 实现、Android 模拟器截图

## 一句话结论

当前家长端已经具备材料导入、听写下发、错题复习、任务状态和 OCR 确认的功能雏形，但信息架构仍像“服务配置页 + 长表单”。按 PRD 定位，家长端的首要价值应是高频作业供给和可追溯确认，因此需要把“布置/导入/确认/追踪”前置，把连接配置降级为状态与设置。

## 当前可取之处

- 已经体现 PRD 的双入口：可以“和 BrainyPal 说需求”，也可以进入结构化导入。
- 高风险动作有候选与确认概念，材料不会直接绕过家长确认下发。
- 任务状态、任务类型、求助次数、家长摘要和 OCR 确认状态已经能在家长侧呈现。
- 文案整体围绕学习证据和任务摘要，没有默认暴露孩子普通聊天原文。

## 主要 UX 风险

1. 首屏层级倒置  
   家长每天最常做的是导入材料、下发任务、看待确认，但当前首屏主要是 Base URL、API Key 和 PIN。

2. 家长入口过低且过于泛化  
   儿童首页底部入口显示为“管理”，更像设置入口。家长侧高频任务应被表达为“给孩子布置任务”“查看待确认”或“家长工作台”。

3. 单页长滚动承载太多工作  
   服务连接、工作台、材料导入、听写、错题、任务状态、OCR 确认都堆在一个页面里，家长难以扫读状态，也难以形成稳定路径。

4. 导入从表单开始，不符合家长心智  
   家长通常从来源开始：拍照、粘贴、历史错题、告诉 AI 教材线索。当前先要求标题、科目、文本，像后台录入。

5. OCR 确认证据不足够可见  
   “查看照片区域/查看裁剪区域”目前在家长状态行里更像文字，不像强证据入口。确认结果会进入学习事实，应优先看到原图高亮区域或裁剪缩略图。

6. OCR 归因按钮效率低  
   “孩子写错了 / OCR 识别错了 / 照片不清楚 / 其实是对的”四个按钮纵向等权堆叠，适合少量确认，不适合批量处理。

7. 手动刷新过多  
   工作台、错题复习、任务状态都依赖显式刷新。家长需要的是“我打开就知道现在怎样”，手动刷新应作为辅助。

8. 视觉气质偏儿童端  
   大卡片、超大标题和圆角适合孩子端亲和感，但家长端更需要安静、紧凑、可扫描的工作台。

## 推荐信息架构

家长端工作台建议拆成四个顶层区域：

| 区域 | 作用 |
| --- | --- |
| 今日供给 | 拍照扫描、粘贴/手输、历史错题、让 AI 找材料。 |
| 待确认 | 候选材料、低置信 OCR、来源/版本确认、确认并下发。 |
| 任务状态 | 已下发、待开始、进行中、卡住、待批改、已完成。 |
| 策略与摘要 | 孩子 AI 使用摘要、阶段目标、提示强度、隐私说明、连接设置。 |

连接配置应变成紧凑状态条：`BrainyPal 已连接` + `设置`。只有未配置或连接失败时，才把配置修复提升到首屏。

## 首轮实现建议

### 1. 工作台首屏重构

- 首屏标题从“家长区域”转为“今天要给孩子准备什么？”或“父母工作台”。
- 前置四个入口：`拍照扫描`、`粘贴材料`、`错题复习`、`问 AI 找材料`。
- 增加状态 chips：`待确认材料`、`待确认 OCR`、`进行中任务`。
- 服务连接折叠到底部或设置动作里。
- 儿童首页家长入口文案改为任务导向。

### 2. 导入入口模式化

- 先选择来源，再进入对应表单。
- `粘贴材料` 复用现有标题/科目/文本能力。
- `快速听写` 作为粘贴材料下的快捷路径，不再单独占一个大块。
- `拍照扫描` 和 `问 AI 找材料` 先可显示为能力入口和依赖状态，等 Agent 能力完整后接入。
- 候选材料卡必须能确认、编辑、确认并下发。

### 3. OCR 证据确认卡片

- 每条待确认 OCR 显示缩略图或裁剪区域。
- 明确展示识别文本、置信度、期望答案或材料项。
- 点击可打开原图高亮区域。
- 归因动作放在证据之后，可采用紧凑按钮组或底部确认 sheet。
- 已确认项折叠展示，避免批量确认时噪音过大。

### 4. 状态与刷新模型

- 进入家长端自动刷新工作台摘要。
- 保留下拉/按钮手动刷新，并显示最近更新时间。
- 任务按状态分组，而不是单列表堆叠。
- 待确认事项应优先于已完成事项。

## 视觉方向

- 保留 BrainyPal 的 navy + cyan 主识别，amber 用于“需要确认/待处理”。
- 家长端减少 hero 字号和大圆角，采用更紧凑的 section、chip、list row。
- 工具入口使用图标：camera、paste、history、search。
- 每个区域只保留一个强主按钮，次级操作降噪。

## 后续 issue 拆解建议

- P1：家长工作台首屏重构与连接设置降级。
- P1：家长导入入口模式化，覆盖拍照/粘贴/历史错题/AI 找材料的入口框架。
- P1：OCR 证据确认卡片与原图区域预览。
- P1：任务状态按待确认/进行中/已完成分组，并减少手动刷新依赖。
- P2：策略与摘要页，包括 AI 使用摘要、阶段目标和提示强度设置。
